Methodology

Data collected from the GitHub API via PyGithub, covering activity in the 180 days prior to 2026-05-19 across the following repositories: anemoi, anemoi-core, anemoi-datasets, anemoi-inference, anemoi-transform, anemoi-utils.

  • Issues: count of issues opened, grouped by the organisation of the issue author.
  • Pull Requests: count of merged PRs where an organisation had at least one contributor. Contributors are identified as: the PR opener, any commit author, and any Co-authored-by: trailer entries in commit messages (resolved via GitHub's noreply address format or a manually maintained email mapping). Each organisation is counted at most once per PR, even if multiple members contributed.
  • Total Reviews: all review submissions on PRs created in the period, grouped by the reviewer's organisation.
  • Unique Reviews: as above, but each reviewer is counted at most once per PR.
  • Merged PR Types: the same merged PR counts above broken down by Conventional Commits prefix (feat, fix, refactor, docs, etc.) parsed from the PR title. PRs whose title does not match a known prefix are grouped as other.
  • Trends Over Time: each run of the data-collection script archives a dated snapshot under history/. The trend figure plots the per-org totals (summed across all repositories) of each metric against the snapshot date, so you can see how contribution patterns evolve. Note that each snapshot is a rolling 6-month window, so adjacent points share overlapping data.
  • Browse History: each archived snapshot is also rendered as a standalone page showing the data as it was at that point in time.

Organisation attribution is based on a manually maintained mapping of GitHub usernames to organisations which may not be exhaustive.
